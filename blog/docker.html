Docker and Docker Compose

Docker is an aplication that simplify the way of manager content of containers. Containers allow a developer to package up an application with all of the parts it needs, such as libraries and other dependencies, and ship it all out as one package. Docker is a bit like a virtual machine but Docker allows applications to use the same Linux kernel as the system that they're running on and only requires applications be shipped with things not already running on the host computer. It gives a significant performance boost and reduces the size of the application.

Requirements

  • Ubuntu 18.04

Start

  • Create an instance (e.g. OpenStack) with Ubuntu 18.04.
  • Follow steps to install and configure Docker.

How to install and configure Docker on Ubuntu 18.04

  • Update official repository of Ubuntu
    1. sudo apt update
  • Install some packages to allows apt use HTTPS
    1. sudo apt install apt-transport-https ca-certificates curl software-properties-common
  • Add GPG key to official repository on your system
    1. curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -
  • Add Docker repository to APT sources
    1. apt-repository "deb [arch=amd64] https://download.docker.com/linux/ubuntu bionic stable"
  • Update the system with Docker repository recently added
    1. sudo apt update
  • Check if you will install from Docker repository rather than from default Ubuntu repository
    1. apt-cache policy docker-ce
  • Install Docker
    1. sudo apt install docker-ce
  • Check if Docker is running
    1. sudo systemctl status docker

Access over SSH (configure in your system)

  1. Add your public key to other machine
  2. docker -H ssh://NAME@IP

Install Docker Compose on your system and instance created

  1. udo curl -L https://github.com/docker/compose/releases/download/1.24.1/docker-compose-`uname -s`-`uname -m` -o /usr/local/bin/docker-compose
  2. sudo chmod +x /usr/local/bin/docker-compose
  3. docker-compose --version

Create docker compose in your system

  1. mkdir docker
  2. vim docker-compose.yml
  3. docker images
  4. docker-compose up

Notes

If you turn off the docker:

  1. docker-compose down

If you turn on the docker:

  1. docker-compose up

When you turn off the docker and turn on again, the configuration files will run again. If you change configuration files (e.g to install new software) you must turn off the docker. But if you update to new versions of software on configuration files, the oldest version persists in docker but it isn't used.
